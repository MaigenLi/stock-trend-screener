## 🎯 新功能详情

### 1. 板块信息获取
#### 数据来源
1. **新浪财经 API** - 获取行业、概念、地区信息
2. **东方财富网页** - 解析板块信息
3. **名称推断** - 从股票名称关键词推断板块
4. **内置热度映射** - 预设板块热度评分

#### 热度评分系统
- **🔥🔥 高热 (80-100分)**: 人工智能、芯片、新能源等
- **🔥 中热 (60-79分)**: 白酒、医药、消费电子等
- **♨️ 温热 (40-59分)**: 银行、证券、化工等
- **⚪ 一般 (0-39分)**: 传统行业、冷门板块

## 🚀 使用方式

### 1. 使用 screener.py（命令行）
```bash
# 全市场扫描，自动生成3个文件
python screener.py --all \
  --price-above ma5 \
  --min-three-day 3 \
  --max-three-day 30 \
  --min-volume-ratio 1.0 \
  --min-score 70 \
  --workers 20
```

### 2. 使用 full_scan.py（交互式）
```bash

python full_scan.py
```

## ⚙️ 技术实现

### 板块信息获取流程
```python
# 1. 检查缓存
cached = load_from_cache(code)
if cached: return cached

# 2. 尝试新浪财经
info = get_from_sina(code)

# 3. 尝试东方财富
if not info:
    info = get_from_eastmoney(code)

# 4. 名称推断
if not info:
    info = infer_from_name(code, name)

# 5. 分析热度
analysis = analyze_sectors(info['sectors'])

# 6. 保存缓存
save_to_cache(code, result)
```

### 缓存机制
- **缓存目录**: `core/sector_cache/`
- **缓存时间**: 24小时
- **缓存键**: MD5哈希前8位
- **文件格式**: JSON

## 💡 使用建议

### 1. 网络连接
- 首次运行需要网络连接获取板块信息
- 后续运行使用缓存，无需网络
- 可离线使用（使用名称推断）

### 2. 性能考虑
- 板块信息获取会增加扫描时间
- 建议使用 `--workers` 参数控制并发
- 缓存机制减少重复请求

### 3. 结果利用
- **代码列表文件**: 导入其他分析工具
- **板块分类文件**: 按板块分析投资组合
- **详细结果文件**: 深入研究个股

## 🔧 故障排除

### 常见问题
1. **板块信息显示"未知"**
   - 检查网络连接
   - 查看 `core/sector_cache/` 目录
   - 运行 `python quick_test_sector.py` 测试

2. **文件生成失败**
   - 检查 `results/current/` 目录权限
   - 确保磁盘空间充足

3. **扫描速度变慢**
   - 减少 `--workers` 参数
   - 清理 `core/sector_cache/` 目录
   - 使用 `--no-sector` 参数（如果添加）
