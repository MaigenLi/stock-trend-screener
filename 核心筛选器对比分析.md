## ⚠️ 注意事项

### 1. **数据一致性**
- 两个工具使用相同的数据源
- 结果应该完全一致（相同参数下）
- 如有差异，检查参数设置

### 2. **性能考虑**
- 全市场扫描较耗时，建议在非交易时间运行
- 可减少`--workers`参数降低CPU使用
- 结果文件较大，定期清理

### 3. **参数验证**
- 使用前先用少量股票测试
- 参考`参数调节指南.md`
- 根据市场环境调整参数

## 🎯 最终结论

### 核心发现
**screener.py 和 full_scan.py 使用完全相同的筛选策略和算法**，主要区别在于：

1. **screener.py** = **算法引擎** + **命令行界面**
   - 优点：灵活、可编程、参数精细
   - 缺点：需要记忆参数

2. **full_scan.py** = **用户界面** + **预设参数**
   - 优点：易用、交互式、快速开始
   - 缺点：参数调节有限

### 推荐使用方案

#### 对于大多数用户
```bash
# 开始阶段：使用 full_scan.py 熟悉系统
python full_scan.py  # 选择2（中等参数）

# 熟练后：使用 screener.py 精细调节
python screener.py --all --price-above ma5 --min-three-day 3 --min-volume-ratio 1.0
```

#### 对于开发者或高级用户
```bash
# 直接使用 screener.py，完全控制
# 或在自己的代码中导入 TrendVolumeScreener 类
```

### 一句话总结
**screener.py 是"发动机"，full_scan.py 是"方向盘"** - 发动机相同，只是操作方式不同。
