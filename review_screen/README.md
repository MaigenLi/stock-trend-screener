# 复盘选股系统

基于实战经验的趋势 + 量能选股系统。

## 核心设计理念

**本质是资金行为**：趋势和量能是一体两面，核心是看有没有真实大资金在里面运作。

**实战判断标准**：
- 涨的时候放量，回调的时候缩量
- 回调不破重要支撑（5日线、10日线）
- 走势流畅，不是急拉急砸
- 有板块联动，不是单打独斗
- 启动位置相对合理（不是已经涨了100%之后）

## 系统结构

```
review_screen/
├── data_cache.py   # 前复权数据缓存（~/.cache/qfq_daily/）
├── indicators.py   # 技术指标计算
├── filter_rules.py # 筛选规则
├── scorer.py       # 评分系统
└── screen.py       # 主入口（唯一参数：--date）
```

## 筛选条件

### 趋势条件
- `MA20 > MA60` — 中期趋势向上
- `收盘 > MA20` — 短期强势
- `DIF连续2日上涨` — 动量向上
- `DEA上涨` — 确认多方
- `MACD连续2天红柱` — 多方市场确认

### 量能条件
- `换手率 ≥ 3%` — 资金活跃
- `量比 ≥ 1.3` — 当日放量
- `20日均成交额 ≥ 1亿` — 流动性保障

### 涨幅条件
- `5% ≤ 3日涨幅 ≤ 22%` — 启动阶段，不过热
- `当日涨幅 > -3%` — 无大跌
- `20日涨幅 < 50%` — 位置不太高

### 风险过滤
- `RSI < 82` — 不过热
- 红柱区间上涨日占比 ≥ 60% — 质量保证

## 评分系统（100分）

| 维度 | 最高分 | 说明 |
|------|--------|------|
| DIF强度 | 30 | 开口大小，多空力量对比 |
| 红柱新鲜度 | 25 | 天数越少越高分 |
| 量能状态 | 20 | 换手率 + 量比 |
| 均线质量 | 25 | 多头排列程度 |

## 用法

```bash
# 复盘指定日期
python screen.py --date 2026-04-22

# 指定股票代码（跳过全市场）
python screen.py --date 2026-04-22 --codes sh600036 sz000001

# 指定线程数
python screen.py --date 2026-04-22 --workers 16
```

## 输出

- 结果文件：`~/stock_reports/review_screen_YYYY-MM-DD.txt`
- 显示：评分/红柱天/收盘/涨幅/换手率/量比/RSI/MA20/MA60
