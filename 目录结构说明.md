## 当前文件说明

### 1. 核心运行文件
- **screener.py**：主筛选器，支持命令行参数调节
- **full_scan.py**：全市场扫描，交互式选择参数
- **full_scan_auto.py**：全市场扫描，自动运行无需交互
- **quick_test.py**：快速功能测试

### 2. 核心模块
- **core/trend_volume_screener.py**：筛选器核心逻辑
  - 股票数据读取（通达信格式）
  - 趋势分析、量能分析、评分计算
  - 股票名称获取（本地+网络）
  - 并行处理支持

- **core/stock_sector.py**：板块信息模块
  - 从东方财富网获取实时板块信息
  - 包含热度和人气评分
  - 本地推断和缓存机制
  - 网络请求优化

- **core/final_full_market_scan.py**：全市场扫描实现
  - 多种参数组合（严格/中等/宽松）
  - 交互式参数选择
  - 结果自动保存

### 3. 数据文件
- **stock_names.json**：股票名称数据库（66只常见股票）
- **stock_name_cache.json**：自动生成的股票名称缓存
- **core/sector_cache/**：自动生成的板块信息缓存

### 4. 文档
- **README.md**：项目总说明
- **使用示例.md**：具体使用示例
- **快速使用指南.md**：快速上手指南
- **新功能更新说明.md**：功能更新记录
- **核心筛选器对比分析.md**：筛选器功能对比

## 使用建议

### 1. 日常使用
```bash
# 快速筛选（使用默认参数）
python screener.py

# 全市场扫描（自动版）
python full_scan_auto.py

# 功能测试
python quick_test.py
```

### 2. 参数调节
```bash
# 自定义参数筛选
python screener.py --min-three-day 5.0 --min-volume-ratio 1.2 --min-score 75.0

# 查看帮助
python screener.py --help
```

### 3. 结果查看
- 筛选结果保存在 `results/current/` 目录
- 包含详细结果、代码列表、板块分类三种格式
- 文件名包含时间戳，便于追踪

## 维护说明

### 1. 定期清理
- 缓存文件会自动生成，可定期清理
- 结果文件可归档到 `results/archive/`

### 2. 数据更新
- 股票名称数据库可手动更新 `stock_names.json`
- 板块信息缓存自动更新，有效期1天

### 3. 功能扩展
- 可修改 `core/trend_volume_screener.py` 扩展筛选逻辑
- 可修改 `core/stock_sector.py` 添加新的数据源
- 可创建新的参数组合文件

## 注意事项

1. **数据源依赖**：需要通达信日线数据（路径：`/mnt/d/new_tdx/vipdoc/`）
2. **网络依赖**：板块信息需要网络连接，失败时使用本地推断
3. **缓存机制**：网络数据会缓存，减少重复请求
4. **并行处理**：默认15线程，可根据系统性能调整

现在目录结构清晰，功能完整，便于使用和维护。
